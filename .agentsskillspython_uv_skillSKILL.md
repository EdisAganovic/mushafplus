
## Translation Updates for i18n.js

### Use Case: Adding Missing Translation Keys

```bash
uv run python << 'PYEOF'
# Read existing translations (handles Windows line endings)
with open('js/i18n.js', 'r', errors='ignore') as f:
    lines = f.readlines()

new_line = '  translationLineHeight: "Razmak redova (prijevod)",\n'

# Find where to insert (after arabicLineHeight)
insert_index = None
for i, line in enumerate(lines):
    if 'arabicLineHeight' in line and i < len(lines) - 1:
        # Check if next line is NOT already translationLineHeight  
        if not lines[i+1].startswith('  translation') or 'translationLineHeight' not in lines[i+1]:
            insert_index = i + 1
            break

if insert_index:
    # Insert new line after arabicLineHeight
    lines.insert(insert_index, new_line)
    
# Write back with original line endings preserved
with open('js/i18n.js', 'w') as f:
    f.writelines(lines)
    print(f"✅ Added translation key at line {insert_index+1}")
else:
    print("❌ Could not find insertion point")
PYEOF
```

### Remove Duplicate Translation Line

```bash
# When sed creates duplicates:
sed -i 'N; /^\(.*\)\n.*\1/d' "js/i18n.js"  # Removes duplicate lines
# or manually delete the specific line number:
sed -i '62d' "js/i18n.js"  # Delete line 62
```

### Complete Example from Production Fix

This was used to fix missing `translationLineHeight` key:

1. **Problem**: Validation error for translation key not in i18n.js
2. **Solution**: Added new key after similar key (`arabicLineHeight`) using sed:
   ```bash
   sed -i 's/^\([[:space:]]*arabicLineHeight: .*\),$/\1,\n  translationLineHeight: "Razmak redova (prijevod)",/' js/i18n.js
   ```

3. **Cleanup**: When duplicate was created, removed with:
   ```bash
   sed -i '62d' js/i18n.js  # Delete line number with duplicate
   ```

4. **Result**: ✅ Translation key added to Bosnian translations file

### Best Practices for Translation Updates

- Always check existing keys first: `grep "existingKey" js/i18n.js`
- Use sed for precision: preserves indentation and commas automatically  
- Verify before/with git diff: `git diff HEAD js/i18n.js`
- Check line count after: `wc -l js/i18n.js`
- Look for encoding issues on Windows: use `sed` vs Python for simple text edits
